# ForJb
It is testing task for job application

Согласно заданию я попытался разработать приложение для ведение учета библиотеки.
Для запуска требуется скачать пакет wp, после запустить приложения, база данных подлючена удаленная, все даннные для входа предстаавлены(далее на локальном порте (пример http://localhost:8080/) при неавтаризованном пользователии нужно авторизоваться (login: admin, password: 1234)
У меня почти получилось представить все функции из требующихся в задании.
Но не все они представлены правитльно, далее мои личные неждочеты:
1. Не полностью реализован REST API, так функции удаления книги реализована через get запрос, а не через delete, к тому же изменения кода книги происходит не через fetch запрос.
2. Так же, исходя из целей приложения, требуется чтобы в нем могли работать только авторизированные пользователи, в данной версии нет защиты от неовтаризированного пользователя (при попадании на начальный экран реализована отправка пользователя на страничку для входа, но остальные страницы не защищены).
Я считаю, что эти недочеты одгни из самых критичных из тех что я успел найти самостоятельно.
